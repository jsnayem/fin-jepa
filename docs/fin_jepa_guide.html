What is Fin-JEPA?

Fin-JEPA is a self-supervised representation learning system designed for daily equity time series data. It learns compact 64-dimensional latent representations of stock features by predicting the future latent state from a context window — entirely within latent space, without ever decoding back to raw inputs.

The core insight: instead of reconstructing noisy price data (like autoencoders) or contrasting augmented views (like SimCLR), Fin-JEPA learns by predicting the encoding of future data from past context. This forces the model to discard non-predictive noise and retain only temporally predictive structure.

Key finding: Latent predictions outperform a naive identity (copy-last) baseline by 10.1%, proving daily financial time series contain learnable temporal structure. However, prediction error does not provide a statistically significant trading signal (AUC ≈ 0.50).

Architecture

The system has three core components:

1. PriceEncoder

A per-day feature extractor that maps 11-dimensional raw features into a latent vector. It consists of a linear projection (11 → embed_dim) followed by a 3-layer MLP with LayerNorm, GELU activations, and a 2-layer output projector. Hidden dimension: 128.

2. TransformerPredictor

A causal (autoregressive) Transformer that operates on the sequence of latent embeddings. It uses 4 layers, 4 attention heads, embed_dim=64, MLP scale factor of 4, and dropout of 0.1. The upper-triangular causal mask ensures each position can only attend to past positions, and the output is shifted by one step (predicts next latent).

3. SIGReg (Collapse Prevention)

The Sketched Isotropic Gaussian Regularizer prevents representational collapse — a common failure mode in self-supervised learning where the encoder outputs a constant vector for all inputs. SIGReg projects latents onto 512 random directions and matches their distribution to a standard Gaussian using 17-knot spline-based regularization. Weight: λ = 0.1.

①ctx → Encoder
②tgt → Encoder
③Predictor(ctx_latents)
④MSE(pred, tgt_latents)
⑤+ λ · SIGReg

The total loss is:

loss = pred_loss + λ × sigreg_loss

where pred_loss is MSE between the predicted latents and target encoder latents, and λ = 0.1 controls the strength of the collapse-prevention regularizer.

LeWM Variant (model_lewm.py)

A more sophisticated architecture with AdaLN-zero modulation, SiLU activations, conv1d embedders, batch-normalized projectors, target LayerNorm, and SmoothL1 loss. Uses embed_dim=192, 16 attention heads, 1024 SIGReg projections, and a history window of 3 context steps predicting 1 future step.

Data Shape

After preprocessing, each sample is a 2D array of shape (65, 11) — 65 consecutive trading days with 11 features each. The sliding window uses 60 context steps and 5 prediction steps with a stride of 5. Data is stored as NPY shards (100K sequences per shard) with a meta.json manifest containing normalization statistics.

Scale: Trained on 2M+ samples from 6,230 stocks spanning 2010–2026.

Output

Model Forward Pass

Calling model.forward(ctx, tgt) returns a dictionary:

{
    'emb': z_ctx,        // (B, T, D) latent embeddings — D=64 by default
    'pred_loss': float,  // MSE between predicted and true target latents
    'sigreg_loss': float,// SIGReg collapse-prevention loss
    'loss': float        // Combined loss = pred_loss + λ × sigreg_loss
}

Inference Methods

  • model.encode_batch(batch) — Encodes all timesteps of a batch in one pass
  • model.predict_future(ctx, steps) — Autoregressive rollout: predicts next latent, feeds it back as input

Trained Checkpoints

Each model variant saves to output/arch_<name>/:

  • best.pt — Model state_dict (best validation loss)
  • meta.json — Training metadata: name, embed_dim, encoder layers, predictor layers, SIGReg projections, total parameters, epochs, elapsed time, and best validation loss

Experiment E Output

Running experiment_e.py produces output/exp_e_results.json:

  • Error statistics: Mean and standard deviation of k=1, k=5, k=20 step prediction errors
  • Spearman correlations: Correlation between prediction error and future volatility
  • AUC scores: ROC-AUC for using prediction error as a direction predictor (up/down)
  • Regime analysis: Return and volatility comparison between high-error and low-error periods

Benchmark Output

Running benchmark.py produces output/benchmark_results.json with per-model inference speed: encoding time (ms), full pipeline time (ms), stocks/second, and time to process 5,000 stocks.

How to Use

Setup

# Clone and set up environment
git clone https://github.com/cedwyh/fin-jepa
cd fin-jepa
python -m venv .venv && source .venv/bin/activate
pip install torch numpy einops pandas scikit-learn scipy huggingface_hub

Quick Test

# Run the built-in model test (prints param count, runs forward pass)
python model.py
# Test the LeWM variant
python model_lewm.py

Data Preparation (one-time)

# Download raw data, compute features, normalize, create NPY shards, upload to HF Hub
cd hf
python prepare_dataset.py

Local Training (Ablation)

Requires the chan-jepa sibling project at ~/dev/chan-jepa/data. Uses live Yahoo Finance data.

# Run 15-variant architecture ablation (50 epochs each)
python compare_arch.py

# Extended ablation: encoder types, predictor types, density sweep
python compare_arch2.py

Full-Scale GPU Training (HuggingFace Jobs)

# Submit to HF Jobs (default t4-small GPU)
bash hf/submit_hf.sh sweep1

# Or run manually within hf/ directory:
cd hf

# Multi-variant sweep (8 configs)
python sweep.py --tag sweep1 --variants 4

# Single model training (200 epochs)
python train_jepa.py --dataset cedwyh/fin-jepa-data --embed-dim 64 --epochs 200

# LeWM-style training
python train_lewm.py --tag lewm_v2 --epochs 200

# Two-phase training (Phase 1: encoder + SIGReg; Phase 2: frozen encoder + predictor)
python train_lewm_v2.py --phase 1 --tag phase1-exp --D 192 --L 6 --epochs 200
python train_lewm_v2.py --phase 2 --tag phase2-exp --load phase1-exp --epochs 200

Downstream Evaluation

# Run VoE analysis (predictor error as trading signal)
python experiment_e.py

Analysis & Benchmarking

# VoE analysis, factor exposure probing, PCA, t-SNE
python analyze.py

# Inference speed benchmark across all variants
python benchmark.py

Conceptual Comparison

MethodObjectiveDecoding?Collapse Risk
Autoencoders / VAEsReconstruct raw inputYesLow (reconstruction prevents it)
SimCLR / TS2VecContrast augmented viewsNoMedium (negative pairs prevent it)
Masked Modeling (MAE)Reconstruct masked patchesYesLow
Fin-JEPA (this work)Predict future latent from pastNoHigh (uses SIGReg)

Fin-JEPA avoids reconstructing noisy price data (which would waste capacity on unpredictable noise) and instead focuses on predicting in the latent space. The SIGReg regularizer is the critical component that prevents the encoder from collapsing to a constant representation.
